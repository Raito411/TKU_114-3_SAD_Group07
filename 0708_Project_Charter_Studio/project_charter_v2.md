## C. 專案章程第 2 版（Project Charter v2）

### 1. 專案題目（Project Title）

```text
校內二手書面交預約系統
```

### 2. 目標使用者（Target Users）

| 角色       | 主要任務                                                             | 主要痛點                                                                   |
| ---------- | -------------------------------------------------------------------- | -------------------------------------------------------------------------- |
| 買方學生   | 搜尋二手書、查看書籍詳細資料、提出購買需求、確認面交時間與地點。     | 書籍資訊分散在社群平台或聊天群組，搜尋效率低，也不容易確認交易是否仍有效。 |
| 賣方學生   | 刊登二手書資料、管理書籍狀態、回覆購買需求、確認面交安排。           | 書籍刊登資訊容易被訊息洗掉，且需要反覆回覆買方詢問書況、價格與是否已售出。 |
| 系統管理員 | 管理交易資料、處理不符合規範的刊登內容或買賣方資格、下架已完成交易。 | 若缺少管理機制，可能出現資料不完整、重複刊登或不符合規範的交易資訊。       |

### 3. 問題陳述（Problem Statement）

```text
對於校內想購買二手書的學生、想出售二手書的學生與負責維護資料品質的系統管理員而言，目前校園二手書交易多分散在社群平台或聊天群組中，常出現書籍資訊不集中、交易狀態不明確、面交時間需要反覆私訊確認、已售出或已預約書籍仍被重複詢問等問題，可能造成學生找書效率低、賣方回覆負擔增加、交易延誤或交易取消。

此外，若缺少基本管理與資料檢查機制，可能出現資料不完整、重複刊登、不符合規範的交易資訊，或讓非交易當事人看到面交時間與地點等敏感資訊。本專案希望設計一個校內二手書面交預約系統，協助賣方集中刊登書籍資料，讓買方依書籍類別或關鍵字搜尋需要的教材，並協助買賣雙方確認面交時間與更新交易狀態。本期聚焦於使用者註冊登入、書籍刊登、書籍搜尋、購買需求提出、面交預約、交易狀態追蹤、資料庫保存與基本管理，不處理線上付款、跨校交易、物流寄送、正式學校帳號登入、付費服務與正式部署。
```

### 4. 專案目標（Project Goal）

```text
本專案目標是讓校內學生能更方便地買賣二手書，降低目前透過社群平台或聊天群組交易時資訊分散、狀態不清楚與面交確認反覆溝通的問題。

系統原型將聚焦於使用者註冊登入、書籍刊登、書籍搜尋、購買需求提出、面交預約、交易狀態追蹤與資料庫保存，並加入基本管理與隱私限制，使買賣雙方可以清楚掌握交易進度，系統管理員也能維護基本資料品質。
```

### 5. 利害關係人摘要（Stakeholder Summary）

```text
本專案主要利害關係人包含買方學生、賣方學生與系統管理員。買方學生關心能否快速找到符合課程或書名需求的二手書，並確認交易是否仍有效；賣方學生關心能否清楚刊登書籍資料、避免重複詢問，並順利安排面交；系統管理員關心刊登內容是否完整、交易狀態是否正確，以及不符合規範的資料是否能被處理。

其他受影響利害關係人包含交易雙方以外的校內學生、學校或平台規範管理者、課程教材資訊提供者。非交易當事人可以瀏覽公開書籍資訊，但不應看到他人面交時間與地點；學校或平台規範管理者關心個資保護與校內交易安全；課程教材資訊提供者可能影響書籍類別、課程名稱或教材版本是否正確。
```

### 6. 範圍內項目（In-Scope）

1. 買方可依據書籍類別、書名或關鍵字搜尋二手書。
2. 賣方可刊登書籍資料，包含書名、類別、價格、書況、照片與交易狀態。
3. 買方可查看書籍詳細資料並提出購買需求。
4. 買賣雙方可確認面交時間與地點。
5. 系統可顯示並更新交易狀態，例如待交易、交易預約中、交易完成。
6. 系統管理員可處理不符合規範的刊登內容，並下架已完成或不適合保留的交易資料。
7. 系統可使用資料庫保存使用者、書籍刊登、購買需求、交易狀態與通知資料。
8. 系統可提供自建使用者註冊與登入，學生帳號可同時買或賣二手書。

### 7. 範圍外項目（Out-of-Scope）

1. 不開放跨校交易。
2. 不支援線上付款或第三方金流服務。
3. 不支援手機原生 App。
4. 不做自動估價。
5. 不處理交易糾紛仲裁或賠償。
6. 不串接學校正式單一登入、正式學校帳號或真實身分驗證系統。
7. 不處理物流寄送、配送追蹤或跨校配送。
8. 不購買或串接付費服務，例如簡訊通知、正式伺服器或正式部署服務。

| 2. 買方搜尋並提出購買需求         | 買方學生           | 買方想找指定書名或類別的二手書   | 買方送出購買需求                   | 買方能依書名或類別搜尋，查看詳細資料，並完成購買需求送出。     |

### 9. 初步成功標準（Initial Success Criteria）

1. 買方可以依書名、類別或關鍵字搜尋二手書，並查看書籍詳細資料。
2. 賣方可以成功刊登一本二手書，且資料包含書名、類別、價格、書況、照片與交易狀態。
3. 買方可以針對指定書籍送出購買需求。
4. 買賣雙方可以確認面交時間與地點，並更新交易狀態。
5. 系統可以區分交易狀態，例如待交易、交易預約中、交易完成。
6. 面交時間與地點僅能被買方、賣方及系統管理員讀取，非交易當事人只能看到公開書籍資訊。

## D. 三分鐘短講提案稿

```text
我們小組的題目是：校內二手書面交預約系統。

目標使用者是：想買二手書的人，想賣二手書的人。

目前的問題是：二手書的買賣充斥在不同的社群軟體或聊天群組之間，沒有一個統一方便的系統讓大家可以買賣二手書。

本系統希望改善：讓二手書買賣資訊統一，能夠使買方可以快速搜尋符合需求的二手書，賣方可以刊登想賣的二手書。

1. 買方可依據書籍類別、書名或關鍵字搜尋二手書。

3. 買賣雙方可確認面交時間與地點。
4. 系統可顯示並更新交易狀態，例如待交易、交易預約中、交易完成。
5. 系統可使用資料庫保存書籍刊登、購買需求、交易狀態與通知資料。
6. 系統可提供自建使用者註冊與登入，學生帳號可買或賣二手書。

本期範圍外包含：
1. 不開放跨校交易。
2. 不支援線上付款或第三方金流服務。
3. 不支援手機原生 App。
4. 不處理交易糾紛仲裁或賠償。

期末原型會展示三個任務：
1. 賣方新增一本二手書資料
2. 買方搜尋並提出購買需求
3. 買賣雙方確認面交並更新交易狀態

目前最大的風險是：範圍擴大，加入線上付款、App、跨校交易、物流或自動估價。

我們的應對方式是：維持範圍外設定，本期只做刊登、搜尋、購買需求、面交預約與交易狀態追蹤。
```
