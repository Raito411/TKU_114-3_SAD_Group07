## 1. 專案背景

```text
本專案為「校內二手書面交預約系統」。目前校園二手書交易多分散在社群平台或聊天群組中，造成買方搜尋效率低、交易狀態不明確、面交時間需要反覆確認，以及賣方需要重複回覆書況、價格與是否已售出的問題。

本期原型聚焦於書籍刊登、書籍搜尋、購買需求提出、面交預約與交易狀態追蹤，不處理線上付款、跨校交易、物流寄送、正式登入、真實身分驗證或手機原生 App。
```

## 2. 目標角色

| 角色     | 主要任務                                                         | 主要痛點                                                                   |
| -------- | ---------------------------------------------------------------- | -------------------------------------------------------------------------- |
| 買方學生 | 搜尋二手書、查看書籍詳細資料、提出購買需求、確認面交時間與地點。 | 書籍資訊分散在社群平台或聊天群組，搜尋效率低，也不容易確認交易是否仍有效。 |
| 賣方學生 | 刊登二手書資料、管理書籍狀態、回覆購買需求、確認面交安排。       | 書籍刊登資訊容易被訊息洗掉，且需要反覆回覆買方詢問書況、價格與是否已售出。 |

## 3. 第一個可操作切片（First Working Slice）

```text
本次第一個可操作切片為「賣方新增一本二手書資料，買方在列表中搜尋並送出購買需求」。

此切片從賣方輸入二手書資料開始，到買方搜尋、查看書籍資料並送出購買需求結束。資料可使用假資料，不需要正式登入、正式資料庫或正式部署。
```

## 4. 本期範圍內項目（In-Scope）

1. 賣方新增二手書資料。
2. 買方依書名、類別或關鍵字搜尋二手書。
3. 買方查看書籍詳細資料。
4. 買方針對指定書籍送出購買需求。
5. 系統顯示交易狀態，例如待交易、交易預約中、交易完成。

## 5. 本期範圍外項目（Out-of-Scope）

1. 不支援線上付款或第三方金流服務。
2. 不支援手機原生 App。
3. 不串接學校正式單一登入、正式學校帳號或真實身分驗證系統。
4. 不處理物流寄送、配送追蹤或跨校配送。
5. 不處理交易糾紛仲裁或賠償。

## 7. 技術限制（Technical Constraints）

| 限制項目         | 本組設定                                              |

| 前端技術         | HTML、CSS、JavaScript                                 |
| 後端技術         | 不使用後端                                            |
| 資料儲存         | 使用假資料（mock data）或瀏覽器暫存，不使用正式資料庫 |
| 是否需要正式部署 | 不需要                                                |
| 是否允許外部套件 | 暫不使用外部套件                                      |
| 是否允許正式登入 | 不允許                                                |

## 8. 程式碼代理不可自行決定的事項

1. 不可自行新增正式登入、註冊或真實身分驗證流程。
2. 不可自行加入線上付款、金流、物流或跨校交易。
3. 不可自行新增 Project Charter v2 未列出的主要角色。
4. 不可自行改變第一個可操作切片。
5. 不可把模擬資料寫成真實使用者資料。

## 10. 交給程式碼代理的正式任務文字

```text
請依照以下系統分析與設計資料，建立第一個可操作切片。

專案背景：
本專案為校內二手書面交預約系統，用來改善二手書資訊分散、交易狀態不明確與面交確認反覆溝通的問題。

目標角色：
賣方學生：刊登二手書資料。
買方學生：搜尋二手書、查看書籍資料並提出購買需求。

第一個可操作切片：
賣方新增一本二手書資料，買方在列表中搜尋並送出購買需求。

本次要支援的需求編號：
FR-01、FR-03、FR-04、NFR-01

本期範圍內：
新增二手書資料、搜尋二手書、查看書籍資料、送出購買需求、顯示交易狀態。

本期範圍外：
線上付款、第三方金流、手機原生 App、正式登入、真實身分驗證、物流寄送、跨校交易。

技術限制：
使用 HTML、CSS、JavaScript 與假資料。不使用後端伺服器、正式資料庫、正式登入或外部套件。

驗收條件：
1. 賣方可以輸入書名、類別、價格、書況、照片說明與交易狀態。
2. 新增後的書籍會出現在書籍列表中。
3. 買方可以依書名、類別或關鍵字搜尋書籍。
4. 買方可以查看書籍詳細資料。
5. 買方可以針對指定書籍送出購買需求，系統顯示送出成功或待賣方回覆狀態。

不可捏造事項：
不可新增正式登入、金流、物流、跨校交易、手機 App 或 Project Charter v2 未列出的主要角色。

請先產生實作計畫，確認後再開始建立或修改檔案。
```
