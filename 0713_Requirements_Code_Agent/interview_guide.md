## 角色 1

| 欄位           | 內容                                                                   |
| -------------- | ---------------------------------------------------------------------- |
| 受訪角色       | 買方學生                                                               |
| 角色與系統關係 | 直接使用系統搜尋二手書、查看書籍資料、提出購買需求與確認面交資訊。     |
| 訪談目的       | 確認買方目前找書流程、痛點、搜尋條件、交易狀態需求與面交資訊隱私限制。 |

| 題號 | 訪談問題                                                         | 想確認的需求或風險               |
| ---- | ---------------------------------------------------------------- | -------------------------------- |
| 1    | 你目前通常怎麼找校內二手書？會在哪些平台或群組搜尋？             | 確認目前流程與資訊分散痛點。     |
| 2    | 你找二手書時最常用哪些條件，例如書名、類別、課程名稱或關鍵字？   | 確認搜尋欄位與資料需求。         |
| 3    | 你如何判斷一本書是否還能購買或已經被預約？                       | 確認交易狀態顯示需求。           |
| 4    | 如果你送出購買需求後，通常希望多久知道賣方是否願意交易？         | 確認購買需求回覆與狀態追蹤需求。 |
| 5    | 面交時間與地點資訊，你認為哪些人可以看到？哪些人不應該看到？     | 確認個資與隱私限制。             |
| 6    | 如果搜尋不到想要的書，或看到的書籍資料不完整，你通常會怎麼處理？ | 確認例外情況與資料完整性風險。   |

## 角色 2

| 欄位           | 內容                                                               |
| -------------- | ------------------------------------------------------------------ |
| 受訪角色       | 賣方學生                                                           |
| 角色與系統關係 | 直接使用系統刊登二手書、管理書籍狀態、回覆購買需求與確認面交安排。 |
| 訪談目的       | 確認賣方刊登流程、必要欄位、重複詢問痛點、面交安排與狀態更新需求。 |

| 題號 | 訪談問題                                                         | 想確認的需求或風險                   |
| ---- | ---------------------------------------------------------------- | ------------------------------------ |
| 1    | 你目前通常如何刊登想出售的二手書？會放在哪些資訊？               | 確認刊登流程與必要欄位。             |
| 2    | 買方最常反覆詢問哪些問題，例如書況、價格、是否已售出或面交方式？ | 確認賣方痛點與資料完整性需求。       |
| 3    | 一本書被預約或售出後，你目前如何更新狀態？                       | 確認交易狀態更新需求。               |
| 4    | 你安排面交時間與地點時，最容易出現哪些溝通問題？                 | 確認面交預約流程與例外情況。         |
| 5    | 如果有多位買方同時提出購買需求，你會如何決定交易順序？           | 確認本期是否需要處理排序或規則。     |
| 6    | 哪些管理規則會讓刊登流程太麻煩，反而降低使用意願？               | 確認基本管理功能不能過度複雜的限制。 |
