# 模擬訪談紀錄（Simulated Interview Notes）

| 受訪角色   | 問題摘要                               | 回答摘要                                                                                                 | 追問                                     | 可能需求                                                                           | 不確定處                                         |
| ---------- | -------------------------------------- | -------------------------------------------------------------------------------------------------------- | ---------------------------------------- | ---------------------------------------------------------------------------------- | ------------------------------------------------ |
| 買方學生   | 你目前如何找二手書？最常遇到什麼問題？ | 通常會在班級群組、社群貼文或學校相關社團搜尋，但資訊分散，常常不知道書是不是已經賣掉。                   | 你搜尋時最常用書名、課程名稱還是類別？   | 系統應支援書名、類別或關鍵字搜尋，並顯示書籍目前交易狀態。                         | 是否需要加入課程名稱作為搜尋條件仍需確認。       |
| 賣方學生   | 你刊登二手書時一定會放哪些資料？       | 通常會放書名、價格、書況、照片和可面交地點，如果有類別或課程資訊也會一起放。                             | 哪些資料一定要填，哪些可以後補？         | 系統應提供賣方新增書籍資料的表單，欄位包含書名、類別、價格、書況、照片與交易狀態。 | 課程名稱是否列為必要欄位仍需確認。               |
| 系統管理員 | 哪些刊登內容需要被下架或處理？         | 如果刊登資料明顯不完整、重複刊登、已完成交易卻仍顯示可交易，或內容不符合校內交易規範，就需要管理員處理。 | 交易完成的資料多久會被處理？會被保存嗎？ | 系統應提供基本管理方式，讓管理員處理不符合規範或已完成的交易資料。                 | 本期只做基本管理，是否需要完整審核流程仍需確認。 |

## 備註

以上內容為模擬訪談紀錄，用於課堂需求萃取練習；後續若取得實際訪談資料，需再依受訪者回答修正。
