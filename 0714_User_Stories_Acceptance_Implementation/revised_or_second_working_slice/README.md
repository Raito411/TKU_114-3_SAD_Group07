# 修正版第一個可操作切片與第二個可操作切片

專案：校內二手書面交預約系統

## 操作方式

先在 `../formal_database_backend/` 執行 `python3 server.py` 啟動 API，再用瀏覽器開啟 `index.html`。若未啟動 API，畫面仍可用假資料展示，但註冊、登入、資料庫與信箱不會連線。

## 本次支援需求

| 需求編號 | 對應內容 |
| --- | --- |
| FR-01 | 買方可依書名、類別或關鍵字搜尋二手書。 |
| FR-02 | 買方可點選查看詳細資料，並看到書籍資料、賣方指定面交地點與交易狀態。 |
| FR-03 | 買方可針對指定書籍送出購買需求，並取消尚未受到賣方確認的購買需求；只有提出需求的買方本人看到待核准提示，賣方不可對自己刊登的書籍送出購買需求。 |
| FR-04 | 賣方可新增一本二手書資料，並在刊登時指定面交地點；新增時交易狀態預設為未交易。 |
| FR-05 | 賣方可查看同書的購買要求清單，看到要求人姓名與發送要求時間，並自由核准其中一位買方；核准前需選擇面交時間，核准後被核准買方會在信箱收到交易核准與面交時間通知，未被選到買方會收到書籍已被其他人購買通知。 |
| FR-06 | 交易狀態區依身分顯示不同內容：買方看自己交易中的書籍狀態與資料，賣方看自己刊登書籍並更新交易狀態。 |
| FR-08 | 賣方可設定二手書折扣，系統顯示折扣後價格。 |
| FR-09 | 賣方可隨時編輯刊登中的書籍資料。 |
| FR-10 | 賣方可輸入唯一刊登序號找到書籍並管理交易狀態。 |
| FR-14 | 賣方可標記交易失敗，系統通知買方、取消需求、書籍恢復未交易並扣買方信任度；信任度達 0 時停權並取消該使用者刊登與購買需求。 |
| NFR-01 | 書籍資料包含刊登序號、書名、類別、價格、書況、照片說明、賣方指定面交地點與交易狀態。 |
| NFR-02 | 面交時間與地點不顯示給非交易當事人。 |
| NFR-03 | 主要功能以「買方搜尋」、「賣方刊登/編輯」、「面交確認與交易狀態」和「使用者信箱」分頁整理。 |

## 驗收方式

1. 切換到「賣方刊登/編輯」分頁，在表單輸入書名、類別、原價、折扣、書況、照片說明與賣方指定面交地點。
2. 確認畫面即時顯示折扣後價格。
3. 送出後，確認新增書籍出現在右側列表，顯示原價、折扣、折扣後價格，且交易狀態預設為未交易。
4. 在搜尋欄輸入書名、類別、價格或關鍵字，確認列表會篩選。
5. 點選「查看詳細資料」，確認進入詳細資料頁面並顯示書籍完整資訊、刊登序號、賣方指定面交地點、折扣與折扣後價格。
6. 點選「送出購買需求」，確認畫面顯示「已送出，待賣方回覆」。
7. 在賣方尚未核准前確認公開/賣方看到「未交易」，提出需求買方看到「交易預約中」，再點選「取消購買需求」，確認書籍狀態仍為「未交易」。
8. 重新送出購買需求後，切換到「面交確認與交易狀態」分頁，再切換為買方學生，確認可看到自己交易中的書籍狀態與資料。
9. 切換為賣方學生，確認可看到賣方刊登清單，並可輸入面交時間、地點與更新交易狀態。
10. 賣方點選「查看要求」，確認可看到要求人姓名與發送要求時間；選擇其中一位買方、填入面交時間並核准，確認系統自動將交易狀態更新為「交易中」，其他待核准需求自動取消，被核准買方信箱收到交易核准與面交時間通知，未被選到買方收到書籍已被其他人購買通知。
11. 回到買方搜尋列表，確認「賣方已核准」不會顯示在書籍卡片上；待核准提示只會出現在送出購買需求的買方本人畫面。
12. 將交易狀態改為「交易完成」，確認系統顯示交易完成。
13. 以非交易當事人查看詳細資料，確認系統自動判斷身分並不顯示面交時間與地點。
14. 確認買方搜尋列表沒有「編輯刊登資料」按鈕。
15. 確認目前使用者自己刊登的書籍不顯示「送出購買需求」按鈕。
16. 在「面交確認與交易狀態」分頁切換為賣方學生，於賣方刊登清單點選「編輯刊登資料」，確認系統切換到「賣方刊登/編輯」分頁並帶入原資料。
17. 在「面交確認與交易狀態」分頁輸入刊登序號，確認系統帶入指定書籍並可更新交易狀態。
18. 賣方核准需求後點選紅色「交易失敗」，確認買方收到交易失敗通知、購買需求被取消、書籍恢復未交易，且買方信任度扣 1。
19. 若買方信任度扣到 0，確認系統將該買方停權，並取消該使用者刊登書籍與購買需求。

## 範圍外項目

本切片已可串接 `../formal_database_backend/` 的正式資料庫與自建註冊/登入 API；若 API 未啟動，會保留假資料展示。本期仍不包含線上付款、物流、跨校交易、手機原生 App、學校正式單一登入、正式學校帳號或真實身分驗證。折扣不串接付款或實際結帳。
