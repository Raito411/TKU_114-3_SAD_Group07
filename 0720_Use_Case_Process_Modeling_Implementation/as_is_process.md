# As-Is 流程描述

## 現況流程：校內二手書交易未集中化

| 步驟 | 參與者 | 行為 | 產出/狀態 | 問題 |
|---|---|---|---|---|
| A-01 | 賣方學生 | 在社群、聊天室或分散管道刊登二手書資訊 | 分散的書籍貼文或訊息 | 資訊分散，不易搜尋 |
| A-02 | 買方學生 | 在不同管道手動尋找二手書 | 可能找到相似或過期資訊 | 搜尋成本高 |
| A-03 | 買方學生 | 私訊詢問書況、價格、是否仍可交易 | 對話紀錄 | 書籍資料不完整時需反覆確認 |
| A-04 | 賣方學生 | 回覆書籍狀態與交易條件 | 個別回覆 | 同樣問題可能重複溝通 |
| A-05 | 買方學生 | 表示購買意願 | 非正式購買需求 | 交易狀態不清楚 |
| A-06 | 買方學生、賣方學生 | 透過私訊協調面交時間與地點 | 面交約定 | 面交確認反覆溝通 |
| A-07 | 賣方學生 | 手動更新或未更新書籍是否已售出 | 貼文狀態可能不一致 | 其他買方可能看到已無法交易的書 |
| A-08 | 系統管理員 | 缺乏集中資料可檢視或處理 | 無完整交易資料 | 難以處理資料品質、停權或異常交易 |

## 主要痛點

| 痛點編號 | 痛點 | 來源 |
|---|---|---|
| P-01 | 二手書資訊分散，買方不易搜尋 | Project Charter v2 問題說明 |
| P-02 | 交易狀態不明確 | Project Charter v2 問題說明 |
| P-03 | 面交確認需要反覆溝通 | Project Charter v2 問題說明 |

| P-05 | 缺乏集中資料，資料品質與停權處理不易 | FR-07、FR-11、FR-14、BR-03、BR-12 |
