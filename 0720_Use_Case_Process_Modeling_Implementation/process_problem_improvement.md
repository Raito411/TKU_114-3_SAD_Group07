# 流程問題與改善對照

| 問題編號 | As-Is 問題 | To-Be 改善方式 | 對應使用案例 | 對應需求 |
|---|---|---|---|---|
| P-01 | 二手書資訊分散，不易搜尋 | 集中建立書籍列表，支援書名、類別或關鍵字搜尋 | UC-01、UC-04 | FR-01、FR-04 |
| P-02 | 書籍是否可交易不清楚 | 系統顯示未交易、交易預約中、交易中、交易完成等狀態；未核准需求在公開狀態仍為未交易 | UC-03、UC-05、UC-06 | FR-03、FR-05、FR-06、BR-08 |
| P-03 | 面交確認反覆溝通 | 賣方核准買方時同時設定面交時間，並透過信箱通知買方 | UC-05、UC-06 | FR-05、FR-13 |
| P-04 | 面交時間與地點有隱私風險 | 面交資訊只顯示給交易當事人，非交易當事人不可見 | UC-06 | NFR-02、BR-01 |
| P-05 | 賣方難以管理多筆購買需求 | 賣方可查看要求人姓名與發送時間，並從多筆需求中選定一位買方 | UC-05 | FR-05、BR-09、BR-15 |
| P-06 | 交易失敗或不可信使用者缺乏處理規則 | 交易失敗會通知買方、取消需求、扣信任度；信任度為 0 時由系統管理員停權 | UC-08 | FR-07、FR-14、BR-03、BR-11、BR-12 |
| P-07 | 資料缺乏正式保存 | 將書籍、使用者、購買需求、通知與交易狀態納入資料庫 | UC-01 至 UC-09 | FR-11、DR-06 |
