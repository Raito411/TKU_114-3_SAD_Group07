# To-Be 流程描述

## 目標流程：校內二手書面交預約系統

| 步驟 | 參與者 | 行為 | 系統回應 | 對應需求 |
|---|---|---|---|---|
| T-01 | 學生使用者 | 註冊或登入系統 | 建立或驗證使用者帳號 | FR-12 |
| T-02 | 賣方學生 | 新增二手書資料，包含書名、類別、價格、書況、照片說明與面交地點 | 建立書籍資料，交易狀態預設為未交易，產生不重複刊登序號 | FR-04、FR-10、NFR-01、BR-05、BR-07、BR-13 |
| T-03 | 賣方學生 | 設定折扣百分比或編輯刊登中資料 | 顯示折扣後價格並更新書籍資料 | FR-08、FR-09、BR-06 |
| T-04 | 買方學生 | 依書名、類別或關鍵字搜尋書籍 | 顯示符合條件的書籍列表 | FR-01 |
| T-05 | 買方學生 | 查看指定書籍詳細資料 | 顯示書籍資料；不對賣方自己的刊登顯示購買按鈕 | FR-02、FR-03、BR-10 |
| T-06 | 買方學生 | 送出或取消未核准購買需求 | 送出需求時通知賣方；取消需求時不通知賣方；對送出需求者顯示交易預約中；公開狀態仍為未交易 | FR-03、BR-08 |
| T-07 | 賣方學生 | 查看要求人的姓名與發送時間，選擇買方並設定面交時間 | 系統將書籍更新為交易中，寄送核可與面交時間通知；未被選到者收到書籍已被其他人購買通知 | FR-05、FR-06、FR-13、BR-04、BR-09、BR-15 |
| T-08 | 買方學生、賣方學生 | 查看交易狀態與信箱通知 | 依身份顯示相關交易資料與通知內容 | FR-06、FR-13、NFR-02、BR-01 |
| T-09 | 賣方學生 | 完成交易或標記交易失敗 | 完成時由賣方更新為交易完成；失敗時通知買方、取消需求、書籍改回未交易並扣信任度 | FR-06、FR-14、BR-04、BR-11 |
| T-10 | 系統管理員 | 處理信任度歸零使用者 | 停權使用者，取消其刊登書籍與購買需求 | FR-07、FR-14、BR-03、BR-12 |

## 範圍限制

- 範圍內：正式資料庫、正式登入與註冊、書籍刊登與搜尋、購買需求、交易狀態、信箱通知、基本管理與停權規則。
- 範圍外：跨校交易、線上付款、第三方金流、手機原生 App、物流寄送、校方 SSO、真實身分驗證、糾紛仲裁、自動估價、付費服務。
