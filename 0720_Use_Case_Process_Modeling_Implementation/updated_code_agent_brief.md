# 更新後程式碼代理實作說明

## 專案

- 專案名稱：校內二手書面交預約系統
- 前次可操作切片：`0714_User_Stories_Acceptance_Implementation/revised_or_second_working_slice/`
- 本次模型來源：0720 使用案例與流程模型

## 本次需對齊的使用案例

| 使用案例 | 需對齊內容 | 對應需求 |
|---|---|---|
| UC-03 送出或取消購買需求 | 買方可送出需求；送出需求時通知賣方；未核准前可取消；取消需求時不通知賣方；取消後賣方查看要求同步移除該筆需求；賣方自己的刊登不顯示購買按鈕；公開狀態維持未交易 | FR-03、BR-08、BR-10 |
| UC-04 刊登、折扣設定與編輯二手書 | 賣方新增書籍時填寫書名、類別、價格、書況、照片說明、面交地點；狀態預設未交易；折扣以百分比與折扣後價格顯示；賣方可編輯刊登中資料 | FR-04、FR-08、FR-09、NFR-01、BR-05、BR-06、BR-13 |
| UC-05 管理購買要求與面交安排 | 賣方可查看要求人姓名與發送時間；可選擇一位買方並輸入面交時間；被選買方收到接受與面交時間通知；未被選到者收到已被其他人購買通知 | FR-05、FR-06、FR-13、BR-04、BR-09、BR-15 |
| UC-08 標記交易失敗與處理停權 | 賣方可標記交易失敗；系統通知買方、取消需求、書籍改回未交易並扣信任度；信任度為 0 時由系統管理員停權並取消資料 | FR-07、FR-14、BR-03、BR-11、BR-12 |

## 資料需求

| 資料 | 說明 |
|---|---|
| 使用者 | 支援註冊、登入、買方/賣方行為、信任度與停權狀態 |
| 書籍 | 書名、類別、價格、書況、照片說明、面交地點、折扣百分比、折扣後價格、刊登序號、交易狀態 |
| 購買需求 | 書籍、買方、發送要求時間、需求狀態 |
| 通知 | 接受交易、面交時間、未被選到、交易失敗等信箱通知 |
| 交易狀態 | 未交易、交易預約中、交易中、交易完成、交易失敗後重新刊登 |

## 實作限制

- 保留正式登入、註冊與正式資料庫。
- 不加入金流、物流、跨校交易、手機原生 App、校方 SSO、真實身分驗證、自動估價或糾紛仲裁。
- 不加入 Project Charter v2 未列出的主要角色。
- 課程名稱仍待確認，不列為本次必要功能。

## 驗收重點

1. 使用案例流程可在畫面操作中被完整走過。
2. 畫面狀態、資料庫資料、信箱通知與交易狀態一致。
3. 買方、賣方、系統管理員看到的資料符合各自身份。
4. 非交易當事人看不到面交時間與地點。
