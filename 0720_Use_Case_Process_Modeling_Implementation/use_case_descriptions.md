# 使用案例詳細描述

## UC-03 送出或取消購買需求

| 欄位 | 內容 |
|---|---|
| 主要參與者 | 買方學生 |
| 支援參與者 | 系統、賣方學生 |
| 對應需求 | FR-03、BR-08、BR-10 |
| 前置條件 | 書籍已刊登，且買方不是該書籍的賣方 |
| 後置條件 | 購買需求被建立、取消，或因賣方核准其他買方而被取消 |

### 主要流程

1. 買方搜尋並查看指定書籍詳細資料。
2. 系統確認該書籍不是買方自己的刊登。
3. 買方送出購買需求。
4. 系統顯示送出成功。
5. 系統通知賣方有新的購買需求。
6. 在公開列表中，該書籍狀態仍顯示為未交易。
7. 對送出需求的買方，系統顯示交易預約中。

### 替代流程

- 買方在尚未被賣方核准前，可以取消已送出的購買需求；取消需求時不通知賣方，且賣方查看要求同步移除該筆已取消需求。

### 例外流程

- 若買方查看的是自己刊登的書籍，系統不提供購買需求按鈕。
- 若購買需求已被賣方核准，買方不可再取消該需求。

### 通知規則

- 買方送出購買需求時，系統會通知賣方。
- 買方取消尚未核准的購買需求時，不通知賣方。

## UC-04 刊登、折扣設定與編輯二手書

| 欄位 | 內容 |
|---|---|
| 主要參與者 | 賣方學生 |
| 支援參與者 | 系統 |
| 對應需求 | FR-04、FR-08、FR-09、NFR-01、BR-05、BR-06、BR-13 |
| 前置條件 | 賣方進入新增或管理書籍功能 |
| 後置條件 | 書籍資料被新增或更新，並可在列表中被搜尋與查看 |

### 主要流程

1. 賣方進入新增書籍分頁。
2. 賣方輸入書名、類別、價格、書況、照片說明與面交地點。
3. 賣方可設定折扣百分比。
4. 系統依價格與折扣百分比顯示折扣後價格。
5. 賣方送出刊登資料。
6. 系統建立刊登序號，且每筆刊登中的序號互不重複。
7. 系統將新書籍狀態預設為未交易。
8. 新增後的書籍出現在書籍列表中。

### 替代流程

- 賣方可修改自己刊登中的書籍資料。

### 例外流程

- 若資料未符合必填欄位，依 NFR-01 需避免建立不完整的書籍資料。

### 待確認事項

- 課程名稱是否列為搜尋或刊登必填欄位，前一版文件仍標示為待確認。

## UC-05 管理購買要求與面交安排

| 欄位 | 內容 |
|---|---|
| 主要參與者 | 賣方學生 |
| 支援參與者 | 買方學生、系統 |
| 對應需求 | FR-05、FR-06、FR-13、FR-14、BR-04、BR-09、BR-11、BR-15 |
| 前置條件 | 賣方的書籍收到一筆或多筆購買需求 |
| 後置條件 | 交易進入交易中、交易完成、交易失敗後重新刊登，或未被選到的需求被取消 |

### 主要流程

1. 賣方進入交易狀態分頁。
2. 賣方使用刊登序號或刊登清單找到指定書籍。
3. 賣方查看該書籍的購買要求。
4. 系統顯示要求人的姓名與發送要求時間。
5. 賣方選擇一位買方並設定面交時間。
6. 系統將該書籍交易狀態自動更新為交易中。
7. 系統寄送接受交易與面交時間通知到被選買方的信箱。
8. 若同一本書有多筆購買需求，系統取消未被選到的需求，並通知那些買方書籍已被其他人購買。

### 替代流程

- 交易完成後，由賣方將交易狀態從交易中更新為交易完成。

### 例外流程

- 若賣方標記交易失敗，系統通知買方交易失敗，取消該買方購買需求，將書籍狀態改回未交易並重新刊登。
- 若買方因交易失敗被扣信任度，信任度最低可降至 0；達到 0 時，系統管理員會將該使用者停權，並取消其刊登書籍與購買需求。

### 隱私規則

- 面交時間與地點只提供交易當事人查看，非交易當事人不應看到。
